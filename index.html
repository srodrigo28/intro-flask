Contexto

Como o Flask surgiu

Flask é um microframework web em Python. Ele apareceu em abril de 2010 com a proposta de manter um núcleo pequeno, legível e fácil de expandir, sem obrigar o desenvolvedor a começar com uma estrutura pesada.

Leitura guiada

🐍Python no centro: o Flask nasceu para ser uma forma leve de criar apps web sem esconder demais o que acontece no código.

🧩Estrutura enxuta: a proposta inicial era entregar o essencial, deixando o projeto crescer por extensões e não por peso no núcleo.

📚Didático por natureza: por mostrar claramente rota, request, response e template, ele virou uma das portas de entrada mais fortes para web em Python.

Mapa rápido da origem

🚀Abril de 2010: o projeto apareceu publicamente e chamou atenção pela simplicidade.

🧪Experimento leve: a ideia inicial não era virar um bloco pesado, e sim um framework flexível e fácil de entender.

🌍Contexto de origem: ele se consolidou no ecossistema Pocoo, ligado ao trabalho de Armin Ronacher na Áustria, com Viena como referência pública mais fácil de localizar.

Fundamento

O que um framework faz na prática

Infra pronta Request Roteamento Response

Um framework organiza o caminho do request. Ele recebe a URL, encontra a função certa, oferece ferramentas prontas e devolve uma resposta padronizada. Você foca na regra, não em remontar a fundação toda vez.

Memorize assim

📥Recebe: o framework aceita a requisição e organiza os dados técnicos de entrada.

🧭Encaminha: ele encontra a rota certa e conecta sua função ao caminho que o usuário pediu.

📤Entrega: ele empacota a resposta e devolve HTML, JSON ou outro retorno de forma padronizada.

Ligação mental rápida

🛠️O que ele entrega: estrutura reutilizável para request, rota, resposta e integrações.

🚫O que ele evita: você reescrever servidor, mapeamento de URL e tratamento básico toda vez.

🧠Fórmula simples: o framework segura a estrada; você decide o que acontece dentro da aplicação.

app.py 
from flask import Flask

app = Flask(__name__)

@app.get("/")
def home():
    return "Flask recebeu a rota e devolveu uma resposta."

Leitura do exemplo

Cria Liga Entrega

1.`Flask(__name__)` cria a aplicação e abre a estrutura principal.

2.`@app.get("/")` liga a URL a uma função que o framework vai chamar.

3.`return` vira a resposta enviada ao navegador no formato certo.

Como organizar o projeto desde o início

Mesmo em um projeto pequeno, separar app, rotas, controllers e templates deixa o aprendizado mais claro e evita misturar tudo no mesmo arquivo.

Base mínima

2. Ativar o venv e instalar o Flask

Venv + Dependencia

Com o Python confirmado, o ideal é ativar um ambiente virtual antes de instalar qualquer dependência. Assim o projeto fica isolado, limpo e mais fácil de manter.

✨ Preparar ambiente virtual (venv) Clique para ver como criar e ativar antes do Flask

O `venv` cria um ambiente isolado para o projeto. Isso evita conflito entre bibliotecas de estudos diferentes e deixa seu treinamento mais organizado.

criar e ativar venv 
python -m venv .venv

# Windows PowerShell
.venv\Scripts\Activate.ps1

# Linux / macOS
source .venv/bin/activate

🧪Use esse ambiente para instalar apenas o que pertence a este projeto.

🧼Isso reduz bagunça e evita misturar bibliotecas com outros estudos ou apps.

Depois que o `venv` estiver ativo, aí sim entre com a biblioteca principal. O mais seguro é usar `python -m pip`, porque isso garante que o `pip` usado é o do seu interpretador atual.

instalar flask 
python -m pip install flask

📦Esse comando baixa e instala o Flask no ambiente em uso.

🧪Depois disso, você já pode testar um `app.py` mínimo e validar se o framework está pronto.

app.py

Ponto de entrada da aplicação

Centraliza a criação do app e é o primeiro arquivo a observar quando você quiser entender como o projeto começa a rodar.

from flask import Flask
from app.routes.page_routes import page_bp

def create_app():
    app = Flask(__name__)
    app.register_blueprint(page_bp)
    return app

app = create_app()

if __name__ == "__main__":
    app.run(debug=True)

app/controllers/page_controller.py

Regras antes da resposta

Ajuda a tirar peso das rotas e concentra a preparação dos dados antes do template.

from flask import render_template

def home():
    dados = {"titulo": "Pagina inicial"}
    return render_template("home.html", **dados)

def about():
    equipe = ["Rotas", "Templates", "Organizacao"]
    return render_template("about.html", equipe=equipe)
🧭

app/routes/page_routes.py

Entradas e caminhos

É a porta de entrada: define as URLs que o navegador pode chamar e para onde cada uma aponta.

from flask import Blueprint
from app.controllers.page_controller import home, about

page_bp = Blueprint("page", __name__)

@page_bp.get("/")
def home_page():
    return home()

@page_bp.get("/sobre")
def about_page():
    return about()

Como rotas, regra e template se conectam

A rota recebe o caminho, o controller organiza os dados e o template monta a página. Esse trio precisa aparecer junto porque é exatamente isso que o iniciante precisa visualizar.

from flask import Blueprint
from app.controllers.page_controller import home

page_bp = Blueprint("page", __name__)

@page_bp.get("/")

    return home()
